理解 SDF（Structure Data File）文件，可以把它想象成化学界的“PDF”——它不仅存着分子的**样子**（2D 或 3D 坐标），还存着分子的**户口信息**（化学属性、实验数据等）。

SDF 是由 MDL 公司开发的一种格式，本质上是多个 **MOL 文件** 的堆叠，并加上了自定义的数据域。

---

## 1. SDF 文件的四大核心板块

一个典型的 SDF 文件由以下四个部分组成：

### ① 标题行 (Header Block)

前三行通常包含：

- **第一行**：分子名称（如果为空，则这行是空白）。
- **第二行**：生成该文件的程序、日期等元数据。
- **第三行**：注释或说明，可以为空。



### ② 计数行与连接表 (Connection Table, CTab)

这是分子的“骨架”核心：

**计数行 (Counts Line)**：分子有多少个原子、多少根键

- 前两个字段分别是原子数和键数，例如"9 8"表示9个原子和8个键
- 氢原子通常不包括在计数中
- 其他字段用于后续版本的扩展，长度分别为3个字符（前11个字段）和6个字符（最后一个字段）



**原子列表 (Atom List)**：记录每个原子的详细信息

每一行代表一个原子，列出其 X, Y, Z 坐标、元素符号、电荷等。

- X、Y、Z坐标（各10个字符），用于描述分子在二维或三维空间中的位置
- 元素符号（3个字符），例如"O"表示氧、"C"表示碳
- 质量差（2个字符，范围-3到+4），表示原子的质量偏差
- 电荷（3个字符），表示原子的电荷状态
- 10个其他字段（各3个字符），通常很少使用，在Progenesis SDF Studio或Progenesis MetaScope中可以为空



**化学键列表 (Bond List)**：描述原子之间的连接

- 第一个字段：第一个原子的索引
- 第二个字段：第二个原子的索引
- 第三个字段：键类型，例如1表示单键、2表示双键
- 第四个字段：立体化学信息，用于描述键的空间构型
- 包含3个很少使用的3个字符字段，可以为空



### ③ 数据域 (Data Items)

这是 SDF 区别于 MOL 文件的关键。在分子的连接表结束后，会有以 `> <TAG_NAME>` 开头的行：

- 这里可以存放各种自定义信息，比如 `> <IC50>`、`> <LogP>` 或 `> <Vendor_ID>`。
- 紧随其后的一行就是该标签对应的值。



### ④ 结束符 (Termination)

- 每个分子的数据以 `**M END`** 结束。
- 在 SDF 中，不同分子之间使用 `**$$$$`** 作为分隔符。

## 2. 为什么 SDF 这么常用？


| **特点**   | **说明**                                   |
| -------- | ---------------------------------------- |
| **可读性强** | 用记事本（Notepad++ 或 VS Code）就能直接打开看懂结构。     |
| **信息量大** | 除了结构，还能带上活性数据，非常适合机器学习和虚拟筛选。             |
| **通用性**  | 几乎所有的化学信息学工具（RDKit, PyMOL, ChemDraw）都支持。 |
| **批量处理** | 一个 SDF 文件可以包含成千上万个小分子。                   |


---

## 3. 如何打开和处理它？

- **肉眼观察**：推荐使用 **VS Code** 或 **Sublime Text**，配合化学插件可以高亮语法。
- **可视化**：
  - **PyMOL / ChimeraX**：查看 3D 构象。
  - **ChemDraw / MarvinView**：查看 2D 结构图。
- **编程处理**：
  - **Python (RDKit)**：这是药化界的标准工具。通过 `Chem.SDMolSupplier` 可以轻松读取其中的坐标和属性。

---

## 4. 一个避坑小贴士

有些 SDF 是 **2D** 的（$Z$ 坐标全为 0），有些是 **3D** 的（经过构象优化）。如果你是要做分子对接（Docking），一定要检查你的 SDF 是否自带合理的 3D 坐标，否则直接输入对接软件可能会报错。
